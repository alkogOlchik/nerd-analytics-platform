# Классические ML-модели Нёрд-аналитикс

Здесь лежат все классические ML-модели для проекта Нёрд-аналитикс. Каждая задача вынесена в отдельную папку: ноутбуки для обучения и сравнения вариантов, скрипты инференса и подробные README внутри папок.

## `tickets_categories`

Модель классификации итоговой категории тикета (`final_category`) по первому клиентскому сообщению и признакам обращения. Сравниваются вариант только по тексту и вариант с дополнительными полями: продукт, приоритет, число переоткрытий. Для инференса используется `predict_ticket_categories.py`, который возвращает предсказанную категорию и ключевые слова, повлиявшие на решение.

Применяется на стадии создания тикета, где пользователю необходимо выбрать категорию запроса. Если не согласен с результатом модели, может поменять вручную.

## `reviews_models`

Группа моделей для обработки отзывов. Каждая подпапка — отдельная задача со своими ноутбуками, препроцессорами и скриптом предсказания.

### `reviews_models/categories`

Классификация категории отзыва (`final_category`) по тексту комментария. В ноутбуках сравниваются модель только по тексту и модель с дополнительными признаками. Инференс выполняется через `predict_categories.py`, на выходе — колонка `predicted_category`.

Применяется для всех отзывов.

### `reviews_models/ratings`

Предсказание числовой оценки отзыва (`rating`) на шкале от 1 до 5. Используются текст комментария и категориальные признаки: продукт, категория, тональность; модель — CatBoost. Скрипт `predict_ratings.py` возвращает округлённый рейтинг в колонке `predicted_rating`.

Применяется только для отзывов оставленных через чат-бота, не в результате оценки обработки тикета. В последнем случае пользователь выставляет оценку по шкале от 1 до 5 вручную.

### `reviews_models/sentiments`

Классификация тональности отзыва (`sentiment`) по тексту комментария. Помимо итоговой тональности скрипт `predict_sentiments.py` выделяет ключевые слова, характерные для позитивного, негативного или нейтрального класса. В ноутбуках сравниваются варианты с текстом-only и с дополнительными признаками.

Применяется для всех отзывов.

## `time_series`

Прогноз количества тикетов поддержки на неделю вперёд в разрезе пар «продукт × категория». Ноутбук `tickets_time_series.ipynb` строит дневные ряды, подбирает модель через Optuna и сохраняет артефакт; скрипт `tickets_time_series_predict.py` загружает готовый `.pkl` и строит рекурсивный прогноз без повторного обучения. На текущем прогоне лучший результат на holdout показала Ridge-регрессия.

Применяется для построения соответствующего графика Дашборда 6 "Тикеты"
