# NerdAPI — платформа интеграций

**Тип продукта:** API интеграция  
**Описание:** NerdAPI — RESTful API-платформа для разработчиков. Позволяет встраивать функционал продуктов компании в сторонние приложения: обрабатывать платежи, получать аналитику, управлять пользователями и заказами.

**Поддержка через платформу:** Нёрд-аналитика  
**Документация:** `/api/v1/docs` (Swagger UI)

---

## Часто задаваемые вопросы — NerdAPI

### Как начать работу с NerdAPI?
1. Зарегистрируйтесь или войдите в личный кабинет.
2. Перейдите в Настройки → API → «Создать токен».
3. Используйте полученный токен в заголовке: `Authorization: Bearer <токен>`.
4. Полная документация: Swagger UI по адресу `/api/v1/docs`.

### Как добавить нового пользователя через API?
```
POST /api/v1/users
Authorization: Bearer <token>
Content-Type: application/json

{"email": "user@example.com", "role": "client"}
```
Возвращает созданного пользователя с `id`. Обязательные поля: `email`, `role`.

### Как получить API-ключ?
Личный кабинет → Настройки → API → «Создать токен». Доступны два типа:
- **JWT-токен** — действует 24 часа, подходит для временных сессий.
- **API Key** — бессрочный, доступен на тарифах Pro и Enterprise.

### Ошибка 401 Unauthorized при валидном токене
JWT-токены действуют 24 часа. Обновите токен через `POST /api/v1/auth/refresh` или выполните повторный вход. Если используете API Key — проверьте, не был ли он отозван (Настройки → API → список токенов).

### Ошибка 403 Forbidden
Недостаточно прав. Проверьте:
- Роль пользователя, от имени которого сделан запрос.
- Что endpoint доступен в вашем тарифе NerdAPI.
- Что IP-адрес входит в allowlist (если включены IP-ограничения).

### Получаю 429 Too Many Requests
Превышен лимит запросов. Лимиты по умолчанию:
- Тариф Free: 60 запросов/минуту
- Тариф Pro: 300 запросов/минуту
- Тариф Enterprise: без ограничений

Добавьте паузы между запросами или запросите увеличение лимита через тикет «Консультация».

### Ошибка 500 на стабильном endpoint
1. Проверьте статус-страницу сервиса.
2. Повторите запрос через 2–3 минуты.
3. Если ошибка воспроизводится стабильно — создайте тикет «Технический сбой» с примером запроса (без чувствительных данных) и полным телом ответа.

### Как работает rate limiting?
Заголовки ответа содержат информацию об ограничениях:
- `X-RateLimit-Limit` — максимум запросов в минуту
- `X-RateLimit-Remaining` — остаток на текущий период
- `X-RateLimit-Reset` — время сброса счётчика (Unix timestamp)

## Типичные проблемы и решения — NerdAPI

| Проблема | Категория | Быстрое решение |
|----------|-----------|-----------------|
| 401 Unauthorized | Проблема с доступом | Обновить токен |
| 403 Forbidden | Проблема с доступом | Проверить роль и тариф |
| 429 Too Many Requests | Техническое ограничение | Снизить частоту запросов |
| 500 Internal Server Error | Технический сбой | Тикет с примером запроса |
| Неверные данные в ответе | Ошибка в данных | Тикет с endpoint, параметрами, ожидаемым/фактическим ответом |
| Вопрос об endpoints | Консультация | Swagger `/api/v1/docs` |
