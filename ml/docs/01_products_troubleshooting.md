# Руководство по продуктам и устранению проблем

## Продукты платформы Нёрд-аналитика

Платформа обслуживает обращения по шести продуктам компании:

- **Веб-сервис** — основной веб-интерфейс платформы
- **Платёжный сервис** — обработка платежей, тарифы, счета
- **Мобильное приложение** — iOS/Android приложение
- **API интеграция** — программный интерфейс для разработчиков
- **Личный кабинет** — профиль пользователя, история обращений
- **Аналитический модуль** — дашборды, отчёты, выгрузки данных

---

## Веб-сервис

### Технический сбой
**Симптомы:** страница не открывается, ошибка 500, сервис недоступен.

**Что делать:**
1. Очистите кэш браузера (Ctrl+Shift+Delete) и обновите страницу (F5 или Ctrl+R).
2. Попробуйте открыть в режиме инкогнито.
3. Проверьте интернет-соединение.
4. Попробуйте другой браузер (Chrome, Firefox, Edge).
5. Если проблема сохраняется — создайте тикет с указанием браузера и текста ошибки.

### Ошибка в данных
**Симптомы:** статистика не совпадает с выгрузкой, неверные цифры в отчёте.

**Что делать:**
1. Уточните период, за который расходятся данные.
2. Проверьте, применены ли одинаковые фильтры.
3. Сделайте повторную выгрузку через 15–30 минут (возможная задержка обновления).
4. Если данные расходятся систематически — передайте в тикет конкретные числа и период.

### Проблема с доступом
**Симптомы:** не войти в аккаунт, аккаунт заблокирован.

**Что делать:**
1. Воспользуйтесь кнопкой «Забыли пароль?» на странице входа.
2. Проверьте папку «Спам» при ожидании письма восстановления.
3. Убедитесь, что используете правильный email (иногда регистрировались через соцсети).
4. При заблокированном аккаунте — обратитесь в поддержку с указанием email.

## Платёжный сервис

### Вопрос по оплате
**Частые вопросы:** что включает тариф, почему произошло списание, как получить счёт.

**Что делать:**
1. Проверьте историю платежей в личном кабинете → раздел «Платежи».
2. Детали тарифа доступны в настройках подписки.
3. Счёт-фактуру можно скачать в разделе «Документы» личного кабинета.
4. Если списание некорректно — создайте тикет с датой и суммой операции.

### Запрос возврата
**Симптомы:** ошибочный платёж, двойное списание, отмена заказа.

**Что делать:**
1. Возврат оформляется только через тикет в поддержку (кнопка «Создать обращение»).
2. К обращению приложите: дату платежа, сумму, причину возврата.
3. Стандартный срок рассмотрения: 3–5 рабочих дней.
4. Деньги возвращаются на карту, с которой был произведён платёж, в течение 7–14 дней после одобрения.

### Технический сбой платежа
**Симптомы:** платёж завис, деньги списались но статус «не оплачено».

**Что делать:**
1. Подождите 15 минут — платёж может обрабатываться.
2. Проверьте SMS/email от банка о списании.
3. Если деньги списаны, но статус не изменился — создайте тикет с номером транзакции из банка.

## Мобильное приложение

### Технический сбой
**Симптомы:** приложение вылетает, не загружается, зависает.

**Что делать:**
1. Перезапустите приложение: полностью закройте и откройте заново.
2. Перезагрузите устройство.
3. Проверьте обновления: App Store / Google Play → убедитесь что установлена актуальная версия.
4. Освободите место на устройстве (минимум 500 МБ свободного пространства).
5. Переустановите приложение, если ничего не помогло.

### Проблема с уведомлениями / подтверждением оплаты
**Симптомы:** не пришло подтверждение, уведомления не работают.

**Что делать:**
1. Проверьте разрешения приложения: Настройки → Уведомления → [название приложения] → включить.
2. Проверьте папку «Спам» в почте.
3. Убедитесь, что в профиле указан правильный email.
4. При необходимости — запросите повторное письмо в личном кабинете.

### Проблема с доступом
**Симптомы:** не войти, забыли пароль, сессия истекла.

**Что делать:**
1. Нажмите «Войти» → «Не помню пароль» → введите email.
2. Если не приходит письмо — проверьте Спам или попробуйте другой email.
3. Принудительный выход из всех устройств: Настройки → Безопасность → Завершить сессии.

## API интеграция

### Консультация по API
**Частые вопросы:** как добавить пользователя, как настроить интеграцию, какие endpoints доступны.

**Что делать:**
1. Документация API доступна по адресу `/api/v1/docs` (Swagger UI).
2. Аутентификация — JWT Bearer Token, получается через `POST /api/v1/auth/login`.
3. Для добавления пользователя: `POST /api/v1/users` с телом `{"email": "...", "role": "..."}`.
4. При ошибке интеграции — укажите в тикете: endpoint, код ответа, тело запроса/ответа.

### Технический сбой API
**Симптомы:** 500 Internal Server Error, timeout, неожиданные коды ответов.

**Что делать:**
1. Проверьте правильность токена авторизации (Bearer token действует 24 часа).
2. Убедитесь, что Content-Type: application/json в заголовках.
3. Проверьте лимиты запросов (rate limiting): по умолчанию 100 запросов/минуту.
4. Логи ошибок доступны в личном кабинете → Настройки → Журнал API.

### Проблема с доступом
**Симптомы:** 401 Unauthorized, 403 Forbidden.

**Что делать:**
1. 401 — токен истёк или неверный, получите новый через `/auth/login`.
2. 403 — недостаточно прав, проверьте роль пользователя (требуется роль с доступом к endpoint).
3. IP-ограничения: если API-ключ привязан к IP, проверьте allowlist.

## Личный кабинет

### Проблема с доступом
**Симптомы:** не войти, заблокирован, не помню пароль.

**Что делать:**
1. Нажмите «Забыли пароль?» → введите email регистрации.
2. Если регистрировались через Google/GitHub — используйте соответствующую кнопку входа.
3. Проверьте, не истёк ли аккаунт (для корпоративных аккаунтов — обратитесь к администратору компании).
4. При блокировке 3+ неудачных попыток — подождите 30 минут или обратитесь в поддержку.

### Жалоба на сервис / качество обслуживания
**Симптомы:** долгое ожидание ответа, некорректный ответ оператора, проблема не решена.

**Что делать:**
1. В личном кабинете нажмите «Создать обращение» → выберите категорию «Жалоба на сервис».
2. Подробно опишите ситуацию: дату, имя сотрудника (если известно), суть претензии.
3. Официальная жалоба рассматривается в течение 5 рабочих дней, ответ приходит на email.
4. Для срочных вопросов — выберите приоритет «Высокий» при создании тикета.

### Запрос документов
**Симптомы:** нужен акт, счёт-фактура, справка.

**Что делать:**
1. Большинство документов доступны автоматически в разделе «Документы» личного кабинета.
2. Акты выполненных работ формируются ежемесячно до 5-го числа следующего месяца.
3. Если документа нет — создайте тикет с указанием типа документа и периода.

## Аналитический модуль

### Ошибка в данных
**Симптомы:** дублирование записей, расхождение в статистике, некорректные данные.

**Что делать:**
1. Проверьте применённые фильтры: даты, продукт, категория.
2. Данные обновляются с задержкой до 1 часа — подождите и обновите страницу.
3. При систематическом дублировании — укажите в тикете конкретные записи с ID.
4. Выгрузка CSV и данные на экране могут отличаться — убедитесь, что выбраны одинаковые фильтры.

### Технический сбой
**Симптомы:** дашборд не загружается, пустые графики, ошибка при загрузке данных.

**Что делать:**
1. Обновите страницу (F5).
2. Очистите кэш браузера.
3. Уменьшите временной диапазон — слишком большой период может вызывать timeout.
4. Попробуйте экспортировать данные в CSV, если визуализация не работает.

### Запрос документов / выгрузок
**Симптомы:** нужна выгрузка данных за период, акт, отчёт.

**Что делать:**
1. На каждом графике есть кнопка экспорта (PNG, PDF, CSV).
2. Выберите нужный период → нажмите иконку экспорта справа от графика.
3. Для большого объёма данных (>3 месяца) выгрузка может занять несколько минут.
4. Если нужны данные в нестандартном формате — создайте тикет с описанием требований.
